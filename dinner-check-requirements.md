# 晩ごはんチェッカー — 要件定義書

## 概要

家族4人の「今日の晩ごはん、いる／いらない」をWebカレンダーで管理し、LINE Botで料理担当に通知するアプリ。他の家族はBotへのリプライで状況確認。

**コンセプト**: 「いらない日」だけ登録すればOK。デフォルトは全員「いる」。

---

## ユーザーと利用フロー

### ユーザー
- 4人家族（名前は設定画面で変更可能。初期値: パパ、ママ、お兄ちゃん、妹）
- ログイン・認証なし。名前を選ぶだけで誰でも誰の分でも登録可能

### 基本フロー
1. LINEグループに貼られたリンクをタップ → Webカレンダーが開く
2. 自分（または家族）の名前を選ぶ
3. カレンダー上で「いらない日」をタップして登録（トグル式：もう一度タップで解除）
4. 登録・変更があるとLINE Botが料理担当に即通知（プッシュ）
5. 毎朝決まった時間に料理担当へ「今日のまとめ」が届く（プッシュ）
6. 他の家族はBotに話しかけて状況確認（リプライ・無料）

## 機能要件

### F1: Webカレンダー画面（フロントエンド）

#### F1-1: メンバー選択
- 画面上部にメンバーのアイコン/名前を横並びで表示
- タップで選択中のメンバーを切り替え
- 選択中のメンバーが視覚的にわかるようにハイライト

#### F1-2: カレンダー表示
- 月表示のカレンダー（今月 + 翌月まで表示可能。約1か月先まで）
- 前月・翌月への切り替えボタン
- 今日の日付をハイライト
- 「いらない」登録済みの日にマーク（×印やグレーアウトなど）を表示

#### F1-3: 登録操作
- 日付をタップ → 「いらない」に登録（トグル）
- 再タップ → 登録解除（「いる」に戻る）
- 過去の日付は編集不可（表示のみ）
- 複数日の一括選択はv1では不要（1日ずつタップ）

#### F1-4: 今日のステータス表示
- カレンダーの上または下に「今日の晩ごはん」サマリーを常時表示
- 例: 「🍚 今日: パパ・ママ・妹（3人）／ ✗ お兄ちゃん」

#### F1-5: 設定画面
- メンバーの名前を変更できる
- 朝の通知時間を設定できる（デフォルト: 8:00）
- LINE Bot連携の設定案内

### F2: LINE Bot通知

#### F2-1: 朝のまとめ通知（定時配信・プッシュ通知）
- 毎朝設定した時間（デフォルト8:00）に **料理担当1人だけ** に個別送信（プッシュメッセージ）
- グループ送信ではなく1対1トークで送ることで、月のメッセージ数を節約（4通→1通/日）
- メッセージ例:
```
🍽 今日の晩ごはん（5/10 土）

🍚 いる: パパ、ママ、妹（3人分）
✗ いらない: お兄ちゃん

📅 カレンダー → [リンク]
```

#### F2-2: 変更時の即時通知（プッシュ通知）
- 誰かが登録・変更したタイミングで **料理担当1人だけ** に個別送信
- メッセージ例:
```
📝 晩ごはん予定が変わりました

お兄ちゃん: 5/10（土）→ ✗ いらない

🍚 今日いる人: パパ、ママ、妹（3人分）
```

#### F2-3: リプライメッセージ（カウント対象外）
- 家族の誰でもBotに話しかけると、今日の晩ごはん状況を返信
- リプライメッセージはLINEの無料通数にカウントされない
- 対応コマンド例:
  - 「今日」→ 今日の晩ごはん状況を返信
  - 「明日」→ 明日の晩ごはん状況を返信
  - 「今週」→ 今週の晩ごはん状況を一覧で返信
- グループトークでもBotへのメンションや特定キーワードで反応

#### F2-4: LINE Bot仕様
- LINE Messaging APIを使用
- グループトークに参加（リプライ応答用）
- プッシュ通知は料理担当1人への個別送信（メッセージ数節約のため）
- 料理担当のLINE UserIDは設定画面で指定

#### F2-5: メッセージ数の見積もり（無料プラン: 月200通）
- 朝まとめ通知: 30日 × 1人 = 30通/月
- 変更通知: 約15回 × 1人 = 15通/月
- **合計: 約45通/月**（無料枠200通に対して十分な余裕あり）
- リプライメッセージは無制限（カウント対象外）

### F3: データ管理

#### F3-1: データモデル
```
dinner_schedule:
  - member_id: string     # メンバー識別子
  - date: string          # YYYY-MM-DD
  - status: string        # "absent" のレコードがある = いらない、レコードなし = いる
  - updated_at: datetime  # 最終更新日時
  - updated_by: string    # 更新者（メンバー名）

members:
  - id: string
  - name: string          # 表示名
  - icon: string          # 絵文字アイコン
  - sort_order: number    # 表示順

settings:
  - morning_notify_time: string  # HH:MM形式
  - line_group_id: string        # Bot参加グループID（リプライ応答用）
  - notify_user_id: string       # 料理担当のLINE UserID（プッシュ通知先）
```

#### F3-2: データ保持
- 過去のデータは少なくとも3か月保持
- 未来のデータは翌月末まで登録可能

## 非機能要件

### プロジェクト情報

- **リポジトリ名**: `dinner-checker`
- **Vercel URL（想定）**: `dinner-checker.vercel.app`

### 技術スタック（推奨）

| レイヤー | 推奨技術 | 理由 |
|---------|---------|------|
| フレームワーク | Next.js (App Router) | フロント・API・Cronを1プロジェクトで完結 |
| ホスティング | Vercel（無料プラン） | Next.jsとの相性最高、Cron対応、完全無料 |
| データベース | Vercel Postgres or Supabase（無料枠） | サーバーレスと相性が良い |
| 定時実行 | Vercel Cron | 朝の通知用（vercel.jsonで設定） |
| LINE連携 | @line/bot-sdk | 公式SDK |

※ Vercelの無料プランで十分運用可能（家族利用のトラフィック量ならまず超えない）

### パフォーマンス
- スマホのLINE内ブラウザで快適に動作すること
- カレンダーの表示・操作は即座にレスポンス（楽観的UI更新）
- 通知の遅延は変更後30秒以内

### UI/UX
- モバイルファースト（横幅380px〜を想定）
- ワンタップで登録完了（確認ダイアログ不要）
- 温かみのあるデザイン（食卓・ごはんをイメージした暖色系）
- 文字サイズは大きめ（家族全員が見やすく）

### セキュリティ
- 認証なし（家族内利用のためシンプルさ優先）
- URLを知っている人のみアクセス可能（推測しにくいURLにする）
- 個人情報は扱わない（名前程度）

## LINE Bot セットアップ手順（参考）

1. [LINE Developers](https://developers.line.biz/) にログイン（LINEアカウントでOK）
2. 新規プロバイダーを作成
3. 「Messaging API」チャネルを作成
4. チャネルアクセストークン（長期）を発行
5. Webhook URLにサーバーのエンドポイントを設定
6. Botを家族のLINEグループに招待
7. グループIDを取得（Botがグループに参加した際のWebhookイベントから取得）

必要な環境変数:
```
LINE_CHANNEL_ACCESS_TOKEN=xxxxx
LINE_CHANNEL_SECRET=xxxxx
LINE_GROUP_ID=xxxxx          # リプライ応答用のグループID
LINE_NOTIFY_USER_ID=xxxxx    # 料理担当のLINE UserID（プッシュ通知先）
DATABASE_URL=xxxxx            # DB接続文字列
```
